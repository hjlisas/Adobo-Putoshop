> The Introduction

I named this program Adobo Putoshop.  I intended partly to spoof Adobe Photoshop and partly to adopt Adobe’s GUI: the multiple file capability, inner windows which can be tiled, a floating toolbox, and iconified toolbox at the top of the screen.

 

I was successful in creating the desired layout of the GUI in Java using JDesktopPane, JFrames, and a JInternalFrame for the floating toolbox.  However, I discovered a limitation.  JDesktopPane is a lower-level container and has problems containing higher-level containers such as AWT’s Canvas.  This problem was manifested when windows hide their contents when they are selected.  Furthermore, the when tiled windows are placed on top of each other, the first window’s canvas (though inactive) is still on top of the other frames and canvases.

 

This is the reason why I used tabbed panes rather that the originally intended  tile windows.

 

However, the functionality of the program is retained.  It can support multiple files.  There is a toolbox containing the object buttons. The program can open and save files.

 

> The GUI

The upper toolbox has three buttons.  Enumerated from the left, there is the New button, the Open button, and the Save button.  This toolbox is detachable.  The lower toolbox has nine toggle buttons (buttons that retain their value even after releasing the mouse button).  This is also detachable.

 

 

 

>> The Upper toolbox (detachable)

 

>>> New button

The New button creates a new canvas and places it in a new tabbed pane that would appear in the tabbed pane panel below the upper toolbox.  When a new canvas is created, then we can now start creating objects. (see > Creating Objects)

 

>>> Open button

The Open button is used in opening a saved project.  Adobo Putoshop (the program) saves the file with a “.uml” extension. (see > Opening Projects File)

 

 

>>> Save button

The Save button is used in saving a project into a file.  To save a file, choose first from the tabbed pane canvases the file to save.  Then press the Save button. (see > Saving Projects)

 

>> Lower Toolbox

 

>>> Move toggle button

The Move toggle button is selected when changing any UMLObjects in the canvas.  Select this to move an object around the canvas and to change its name, methods, fields, cardinality, etc.

 

>>> Type toggle buttons

These toggle buttons are distinguished by capital letters C, A, and I imposed on coloured backgrounds.  These define Type Objects: UMLObjects that define either a class, an abstract class or an interface.

 

>>>> Concrete Class toggle button

Concrete Classes are UMLObjects that represents actual Java Objects.  They are Concrete as distinguished from “Abstract.”

 

>>>> Abstract Class toggle button

Abstract Classes are classes that leaves definitions for some methods to actual instances.

 

>>>> Interface toggle button

Interfaces makes possible the separation of method enumeration and definition.  It merely defines the methods and fields which an object MUST define but does not define it in itself.  It differs from an Abstract Class in the point that an Abstract Class may define some of its methods and leave the others to be defined by their instances.  Interfaces do not define their methods.

 

>>> Relation toggle buttons

These toggle buttons are distinguished by icons that illustrate the line tips associated with their diagrams.  Relations define the dynamics between two or more classes. 

 

>>>> Extends toggle button

Extension means that the source object instantiates (in cases of a concrete class or abstract class parent) or implements (in cases of interface parents) another concrete, abstract class, or an interface.  This is illustrated by a line with a closed arrow tip of white colour on the end tip.

 

>>>> Association toggle button

Associations are illustrated by a line with no tips.  These are Bi-Relational Relations and thus, takes note of the source and destination when setting its cardinality.

 

>>>> Dependency toggle button

Dependencies are weak forms of relations.  They are illustrated by an open arrow line tip at its end.

 

>>>> Composition toggle button

Composition is represented by a diamond tip of colour black at the end.

 

>>>> Aggregation toggle button

Aggregation is represented by a diamond tip of colour white at the end.

 

 

>> Tabbed Pane Panel

The center of the screen is appropriated for the tabbed pane panel.  This should contain the tabbed panes containing the canvases on use.

 

> Using the Program

 

>> Creating Objects

To create objects, you first must create a canvas.  So, you click on the New button at the upper toolbox, and you'll see a tabbed pane appear at the center of the desktop.  Consecutive presses on this button would create a number of tabbed panes.  The tabbed panes are titled, by default, "Untitled #" where # stands for any integer starting from zero depending on its order of creation.

 

When the desktop is created, then it is now possible to create the UMLObjects.  The selection of UML Objects are available at the lower toolbar.  These buttons are toggle buttons, thus, they retain their value after you press them and as long as you do not press any other toggle button from within that group (lower toolbar button group).  To create the object, just select any of the buttons that correspond to a UMLObject (the C button for a Concrete Class Object, A for an Abstract Class Object, I for an Interface Object, the line tip illustrations for their respective Objects, note however that the line objects can only be created when there are two Class or Interface Objects present).  When the appropriate button is selected, then it is now possible to create the Object on the canvas tabbed pane by clicking once on any part of the canvas.

 

A default Object box with its default name, fields, and methods would appear on the canvas.  Normally it would be a box with three divisions within it.  The top division would correspond to the Object's name, the second would correspond to its fields, and the third would correspond to its behaviour. 

 

>> Editing Objects

You could edit the default values by first, selecting the "Move” button ( + ) at the lower toolbox and double-clicking on the Object.  A dialog box would appear where the default values are presented.

 

 

To edit the name of the Object, a textfield is provided on the upper portion of the dialog box.  By default, the default name is placed.  You may change this value on Type UMLObjects but not on Relation UMLObjects.

 

For the Type UMLObjects, choose first from the field and method tabs the desired section to edit.  The field and method tabbed panes GUI are similar.   They could be edited by entering a method/field name on a textfield the tabs.  Then, press the “Add” button.  This would add the specified name to the list below the textfield.  A number of methods/fields could be added.  The user may select among the elements of this list for editing or deleting.  To select, the user may just point with the mouse on the respective name and click once.  The name would appear in the textfield.  To edit, this said name, then edit the textfield and press “Edit.”  Pressing “Add” would add the edited name to the list.  Pressing “Delete” would delete the name from the list.

 

Special feature: if you enter a pound sign ( # ) before a method or field name, this would italicise the name in the canvas.  This is useful if you want to differentiate this name as abstract.

 

For the Relation UMLObjects, select from the List the methods to edit.  The Method name appears in the textfield when selected but it is not editable.  The only textfield available for editing is the cardinality textfield.  Specify the cardinality of the relation then press Edit.

 

For Bi-Relational UMLObjects, there are two tabbed panes specified: the Source and the Destination Objects.  Select the proper tab and do as the above to edit the cardinality.

 

>> Saving Projects

Choose the project to save by selecting from the tabbed pane panel the appropriate tab of the project.  Press the Save button.  This opens up a FileChooser dialog box.  You can select the desired directory and type in the desired name of the file.  The default extension is “.uml”.  To override this extension default, specify the extension with the file.  Not specifying the extension invokes the default extension to be appended to the filename specified.

 

 

When pressing the “Save” button of the dialog FileChooser, Adobo calls UMLCanvas to save the project.  UMLCanvas merely saves the Vector that contains all the UMLObjects of the said project.  The UMLCanvas backpointer of each object is not saved since it is specified as transcient.

 

>> Open Projects

Press the Open button. This opens up a FileChooser dialog box.  The default root is the c:\windows directory.  One can navigate through the available system resources until the desired file is found.  When the file is selected, the file is loaded by the program.  A new tabbed pane is created together with a new canvas.  The file actually stores a Vector of uml objects.  The program (actually, UMLCanvas) iterates these objects and adds them to the canvas (because canvas is never stored).  As it adds the objects, it sets the backpointer to the canvas and the Object as mouse listener and mouse motion listener.
